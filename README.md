# 🚀 Convention

우리 팀의 원활한 협업과 일관성 있는 코드 관리를 위한 규칙입니다. 모든 팀원은 개발 시작 전에 반드시 숙지해 주세요.

---

## 🌿 Git 브랜치 전략

우리 프로젝트는 **Feature-Driven Workflow**를 따릅니다. **`main` 브랜치로의 직접적인 Push는 절대 금지**합니다.

### 📌 브랜치 명명 규칙
`종류/기능명` 형태로 생성하며, 단어 구문은 하이픈(`-`)을 사용합니다.

| 종류         | 설명                       | 예시                                        |
|:-----------|:-------------------------|:------------------------------------------|
| `feature`  | 새로운 기능 구현                | `feature/post-crud`, `feature/comment`    |
| `fix`      | 버그 및 에러 수정               | `fix/db-connection`, `fix/error-response` |
| `docs`     | 문서 수정 (README, API 명세 등) | `docs/readme-update`                      |
| `refactor` | 코드 리팩토링                  | `refactor/dto-separation`                 |
| `chore`    | 기타                       | `chore/unuse-file-delete`                 |

> ⚠️ **주의:** 브랜치를 생성하기 전에 항상 `main` 브랜치에서 `git pull`을 진행하여 최신 상태를 유지하세요.

## 🧭 Trip 경로 데이터 전환

Trip 생성 시 사용할 경로 제공자는 `TRIP_ROUTE_PROVIDER` 환경변수로 선택합니다.

```text
TRIP_ROUTE_PROVIDER=hybrid
```

- `hybrid`(기본값): 고정 지하철 막차표, 노선별 버스/N버스 목데이터,
  카카오 길찾기의 실시간 택시 정보를 함께 반환합니다.
- `mock`: 지하철, 버스, N버스, 택시 데모 데이터를 반환합니다.
- `real`: 아래 실제 외부 API를 조합해 조회된 경로만 반환합니다.
  - 지하철: 서울교통공사 홍대입구역(`239`) 막차 시간표
  - 일반 버스: 서울 버스 753번(`100100118`) 정류장별 막차 정보
  - 심야 버스: 서울 N62번(`100100588`) 홍대입구역 정류장 막차 정보
  - 택시: 카카오 길찾기의 실제 택시비, 거리, 소요시간

실제 모드는 다음 환경변수를 사용합니다.

```text
KAKAO_REST_KEY=...
SEOUL_METRO_KEY=...
BUS_SERVICE_KEY=...
```

하이브리드 모드에서는 택시 조회를 위한 `KAKAO_REST_KEY`만 필요합니다.
택시 API가 실패하더라도 고정 지하철과 버스/N버스 후보는 반환합니다.

### 하이브리드 막차 카탈로그

출발지에서 홍대입구역과 상수역까지의 직선거리를 비교해 가까운 역의
시간표와 버스 정류장 목데이터를 사용합니다. 목적지가 선택된 역보다
동쪽인지 서쪽인지에 따라 지하철 방향을 정합니다.

| 역 | 동쪽 방향 막차 | 서쪽 방향 막차 |
|:---|:---|:---|
| 홍대입구역 2호선 | 을지로입구행 00:49 | 신도림행 00:50 |
| 상수역 6호선 | 공덕행 00:53 | 새절행 00:42 |

- 위 지하철 시간은 제공된 첫·막차표에서 해당 역을 출발해 이동할 수 있는
  막차만 옮긴 고정 데이터입니다. 홍대입구역 종착 열차는 제외했습니다.
- 별도 주말 시간표가 없으므로 현재는 모든 요일에 같은 시간표를 사용합니다.
- 일반 버스와 N버스의 노선, 막차 시각, 소요시간, 요금은 화면 연동용
  목데이터이며 실제 운행 정보가 아닙니다.
- 카탈로그 값은 `src/main/resources/application.yml`의
  `trip.hybrid.stations`에서 수정할 수 있습니다.

대중교통 API가 좌표 기반 전체 경로 탐색을 제공하지 않기 때문에 MVP에서는 다음 범위로 제한합니다.

- 출발지가 홍익대학교 서울캠퍼스에서 반경 3km 이내일 때만 홍익대 카탈로그를 적용합니다.
- 지하철 방향은 목적지의 동/서 위치를 기준으로 홍대입구역 내선/외선을 선택합니다.
- 753번과 N62번은 목적지 이름·주소가 각 노선의 주요 경유 권역에 해당할 때 조회합니다.
- `scheduledAt`은 실제 외부 API의 막차 시각입니다.
- 버스·지하철 `totalMinutes`와 지하철 거리 추가요금은 좌표 거리 기반 예상치입니다.
- 어느 한 외부 API가 실패하면 해당 후보만 제외하며, 다른 실제 후보를 mock으로 대체하지 않습니다.

## 🖥️ 로컬 프론트 연동 데모

저장소 루트에서 백엔드와 프론트를 각각 실행합니다.

```bash
# 터미널 1: H2 + 하이브리드 경로로 백엔드 실행
SPRING_PROFILES_ACTIVE=local TRIP_ROUTE_PROVIDER=hybrid \
KAKAO_REST_KEY=카카오_REST_API_키 ./gradlew bootRun

# 터미널 2: 최초 한 번
npm install

# 터미널 2: 프론트 실행
npm run dev
```

브라우저에서 `http://localhost:3000`을 엽니다. Vite가 `/api` 요청을
`http://localhost:8080`으로 프록시하므로 별도의 프론트 환경변수는 필요하지 않습니다.

- H2 서버를 다시 시작한 뒤에는 로그인 화면에서 `회원가입 후 로그인`을 먼저 누릅니다.
- `KAKAO_REST_KEY`가 없거나 택시 API가 실패해도 고정 지하철과 버스/N버스
  후보는 정상적으로 반환됩니다.
- 기존 서울 지하철·버스 외부 API 방식은 `TRIP_ROUTE_PROVIDER=real`과
  필요한 API 키를 설정해 별도로 확인할 수 있습니다.
- 타이머 화면 오른쪽의 `DEV TIME CONTROL`에서 골든타임 30·15·5분 전, 정각,
  1분 경과 상태로 즉시 이동할 수 있습니다.
- 테스트 시간은 브라우저 표시와 인앱 알람 판정에만 적용되며 서버·DB 시간은 변경하지 않습니다.
